# 퍼스트 랜딩 검수엔진 고정 정책 (2026-02-24)

## 목적
- 루트 메인 랜딩(`https://dliver.co.kr/`)에서 원고 검수/스코어링/피드백 루프를 제공해 체류시간과 회원 전환율을 높인다.

## 절대 고정값
1. 초기 버전은 **LLM 호출 금지** (OpenAI API 비용 0원 정책)
2. 점수 산정은 **규칙엔진 100%**
3. 허용 파일 형식은 **doc/docx/pdf**
4. 비회원은 **1일 1회 체험**
5. 재점수 및 히스토리 활용은 회원 중심으로 유도
6. 검수 파일/결과 보관은 **30일 후 자동 정리**

## 점수 체계
- 승인 가능성 점수(0~100)
- 리스크 점수(0~100, 높을수록 위험)
  - 법/표시/과장/비방 하위 점수 포함
- 가독성 점수(문장 길이/중복/구조)
- 브랜딩 적합도(톤/일관성)

## 라우팅 고정
- `/` : 신규 퍼스트 랜딩(검수 중심)
- `/self-order` : 기존 셀프주문 랜딩(세컨드 랜딩)

## 운영 주의사항
- OpenAI 관련 키가 설정되어 있어도 퍼스트 랜딩 검수 API에서는 사용하지 않는다.
- 규칙엔진 튜닝은 사전/가중치 파일 변경으로만 진행한다.
- 정책 변경 시 본 문서와 개발원칙 문서를 동시에 갱신한다.
