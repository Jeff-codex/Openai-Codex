# 회원전용페이지 UI 개편 체크리스트

## 목표
- `https://everyonepr.com/member/`의 정보 구조와 시각 위계를 더 명확하게 정리한다.
- 주문 등록, 매체 선택, 결제 확인, 주문 현황 확인 흐름을 한 화면에서 더 직관적으로 만든다.
- 공개 랜딩과 브랜드 톤은 맞추되, 회원 포털은 더 실무형 대시보드답게 정리한다.

## 범위
- 회원전용페이지 UI/UX만 다룬다.
- 대상 파일
  - `01_서비스코드-ServiceCode/회원전용페이지-MemberPortal/index.html`
  - `01_서비스코드-ServiceCode/회원전용페이지-MemberPortal/styles.css`
  - `01_서비스코드-ServiceCode/회원전용페이지-MemberPortal/app.js`
- 제외
  - 결제 API 로직 변경
  - 관리자페이지 UI 변경
  - 도메인/Cloudflare 설정
  - 공개 랜딩 구조 재개편

## 1. 상단 구조
- 상단바 높이, 로고 크기, 상태 pill, 로그아웃 버튼 위계를 다시 조정한다.
- `회원 확인중...` 상태 문구와 로그인 완료 상태 문구를 더 짧고 읽기 쉽게 정리한다.
- 상단바가 모바일에서 2줄로 깨질 때도 높이가 과도하게 커지지 않게 맞춘다.

## 2. 대시보드 요약 영역
- `실시간 운영 대시보드` disclosure를 더 직관적인 카드형 요약 구조로 개선할지 판단한다.
- `누적 결제금액 / 내 총 주문 / 진행 중 주문 / 송출 완료` 4개 지표의 중요도 순서를 점검한다.
- 숫자 강조, 라벨 길이, 카드 간 간격을 재정리한다.

## 3. 매체 선택 UX
- 좌측 `매체 선택` 영역의 검색, 카테고리 nav, 카드 리스트 위계를 재정리한다.
- 카테고리 nav가 스크롤/모바일에서 답답하지 않은지 점검한다.
- 선택된 매체와 매체 목록의 대비를 더 분명하게 만든다.
- 회원 전용 단가/노출채널/설명 블록을 더 읽기 쉬운 구조로 조정한다.

## 4. 주문 등록 UX
- `송출 주문 등록` 카드에서 입력 순서가 자연스러운지 점검한다.
- `주문명`, `요청사항`, `원고 파일 첨부`, `주문 등록` 버튼의 시선 흐름을 정리한다.
- `원고파일첨부` 버튼 문구, 파일명 표시, 힌트 문구의 리듬을 다듬는다.
- 선택 매체 요약 박스의 금액 위계와 문장 길이를 재정리한다.

## 5. 결제 확인 모달
- 결제 모달의 요약 카드, 금액 위계, 환불 규정 배치가 충분히 읽기 쉬운지 점검한다.
- 결제 위젯 영역이 비어 있는 상태와 채워진 상태 모두 시각적으로 어색하지 않게 만든다.
- `취소 / 결제하기` CTA 대비를 더 분명하게 만든다.

## 6. 주문 현황 테이블
- 데스크톱/모바일에서 표 가독성을 점검한다.
- 주문 상태 badge, 결제금액 표기, 첨부 컬럼의 정보 밀도를 조절한다.
- 모바일에서 카드형 fallback이 필요한지 검토한다.

## 7. 카피/라벨 정리
- 회원 포털 안의 버튼/라벨/힌트 문구를 더 짧고 소비자 친화적으로 다듬는다.
- `회원전용`, `주문등록`, `송출`, `결제예정금액` 같은 용어가 과하게 딱딱하지 않은지 점검한다.
- 공개면과 다르게 회원 포털은 설명보다 행동 중심 문구를 우선한다.

## 8. 시각 디자인
- 공개 사이트와 같은 브랜드 시스템을 유지하되, 대시보드답게 더 정돈된 레이아웃으로 만든다.
- 카드 입체감, 그림자, border, 표면 깊이를 다시 맞춘다.
- 숫자/라벨/버튼/상태 badge의 타이포 위계를 다시 설계한다.
- 필요하면 보조 아이콘을 제한적으로 추가한다.

## 9. 검증 체크리스트
- 로그인 후 `https://everyonepr.com/member/` 정상 진입
- `/api/auth/me`, `/api/media`, `/api/orders` same-origin 정상
- 주문 등록 -> 결제 모달 진입 정상
- 로그아웃 후 홈 복귀 정상
- 데스크톱/모바일 UI 확인
- `check:html:guard` 통과
- restart checklist 기준 `DEPLOY_READY=YES`

## 완료 기준
- 회원이 첫 화면에서 `무엇을 먼저 해야 하는지` 바로 이해할 수 있다.
- 매체 선택, 주문 등록, 결제, 주문 현황이 서로 경쟁하지 않고 우선순위가 보인다.
- 공개면보다 더 실무형이고, 관리자페이지보다 더 친화적인 중간 톤으로 정리된다.
